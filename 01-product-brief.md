# 01. 제품 요약: Onestack 차세대 클라우드 배포 플랫폼 홍보 페이지

상태: `EXECUTION_READY`

작성일: 2026-04-17

## 1) 한 줄 설명

**"개발자는 코드에만 집중하고, 보안과 속도는 Onestack이 책임지는 'Zero-Friction' 클라우드 인프라"**

## 2) 문제 정의

### 현 시장의 문제점
- 개발자는 배포 설정(인프라, 보안, CDN, SSL)에 시간 낭비
- CTO/기술리더는 보안(DDoS, WAF, 원점 보호)과 비용(엑스트라 비용)을 동시에 해결해야 함
- 기존 솔루션(Vercel, Heroku)은 제한적 기능 + 고가 요금, 또는 Coolify/Dokploy 같은 자체 호스팅은 운영 부담 과중

### 왜 지금?
- 엣지 컴퓨팅, DDoS 공격, 글로벌 배포 필요성 급증
- 개발팀의 빠른 반복 요구 vs. 운영 복잡성 증가의 괴리
- DevOps 전문가 부족 + 인건비 폭증

## 3) 목표

### 홍보 페이지의 목표
1. **인지**: "Code to Edge, Securely" 컨셉을 통해 Onestack의 차별성 확실히 인식
2. **전환**: 5가지 핵심 가치제안(Zero-Config, Security, Performance, Automation, Cost)으로 방문자 → 유료 고객 유도
3. **신뢰**: Vercel/Heroku와 경쟁하는 수준의 전문성/우수성 시각적·언어적 전달
4. **커뮤니티**: 개발자 커뮤니티 참여 유도 (GitHub Stars, Discord, 베타 신청)

## 4) 비목표 (Non-goals)

- ❌ 기술 스펙 설명서 (마케팅 페이지이지 문서가 아님)
- ❌ 가격 책정 상세 공개 (가격 책정은 별도 페이지)
- ❌ 모든 기능 나열 (5가지 핵심 가치에 집중)
- ❌ 경쟁사 명시적 비난 (비교 금지, 가치 강조만)

## 5) 범위 (v1)

### 페이지 구조 & 섹션
1. **Hero Section**
   - 헤드라인: "Code to Edge, Securely"
   - 서브헤드라인: "복잡한 설정 없이 보안과 속도를 갖춘 차세대 배포"
   - CTA: "지금 시작하기" / "데모 보기"

2. **5가지 가치제안 카드 섹션**
   - ① 개발 생산성의 혁신 (Zero-Config, Preview Deployment, Self-Service)
   - ② 철벽 보안 (DDoS Shield, WAF & Bot Defense, Origin Cloaking)
   - ③ 글로벌 초고속 네트워크 (Edge CDN, Smart Routing)
   - ④ 운영 자동화 (Auto SSL, Full GitOps)
   - ⑤ 탁월한 경제성 (Infrastructure Abstraction, Egress Savings)

3. **사회적 증명 (Social Proof)**
   - 사용자 수, 배포 건수, 고객사 로고
   - 고객 후기/사례

4. **비교 테이블 (선택)**
   - Vercel vs Heroku vs Onestack (기능 비교)

5. **CTA & 다음 단계**
   - "지금 배포 시작하기"
   - "GitHub 저장소" 링크
   - "Discord 커뮤니티 참여"
   - 이메일 뉴스레터 가입

## 6) 사용자/역할

### 주요 페르소나
| 페르소나 | 관심사 | 메시지 | CTA |
|---|---|---|---|
| **개발자** | 배포 속도, 간편함 | Zero-Config, 5분 배포 | 무료로 시작 |
| **CTO/기술리더** | 보안, 비용, 확장성 | DDoS 방어, 글로벌 성능 | 엔터프라이즈 데모 |
| **DevOps 엔지니어** | 자동화, 모니터링, 제어 | GitOps, 실시간 모니터링, API | 문서 & API 참고 |
| **스타트업/중소팀** | 빠른 성장, 저비용 | "비용 70% 절감" | 14일 무료 트라이얼 |

## 7) 성공 지표 (KPI)

### 마케팅 메트릭
1. **페이지뷰 & 체류시간**: Hero 섹션 방문 후 각 가치제안 카드 클릭률 > 40%
2. **CTA 전환율**: "지금 시작하기" 클릭 후 가입 완료 > 15%
3. **소셜 신호**: GitHub Stars 증가, Discord 가입자 증가
4. **SEO**: "클라우드 배포", "서버리스 배포", "DDoS 방어" 키워드 상위 10 진입

### 디자인 메트릭
1. **Lighthouse**: 성능 90+, 접근성 95+, SEO 100
2. **로딩 시간**: First Contentful Paint < 1.5s
3. **모바일 반응성**: 모든 해상도에서 완벽한 레이아웃

## 8) 핵심 가설

1. **가설 1**: 개발자는 "배포 자동화 + 보안 자동화"라는 이중 가치에 강력하게 반응한다.
   - 테스트: CTA 클릭률 추적, 고객 인터뷰

2. **가설 2**: 경쟁사(Vercel, Heroku)와 구별되는 "초고속 네트워크 + 저비용" 메시지가 핵심 차별화다.
   - 테스트: 경쟁사 비교 섹션 A/B 테스트

3. **가설 3**: 시각적 우수성(디자인, 애니메이션)이 전환율을 5~10% 높인다.
   - 테스트: 단순 vs. 고급 디자인 A/B 테스트

## 9) 이해관계자

- **의사결정자**: Onestack 제품 관리자, 마케팅 담당자
- **협업 부서**: 
  - 개발팀 (웹페이지 기술 구현)
  - 디자인팀 (UI/UX, 브랜드 일관성)
  - 마케팅팀 (메시지, SEO, 분석)
  - 기술 콘텐츠팀 (사용 사례, 비교 테이블 검증)

## 10) 오픈 질문

- QUESTION-001: 페이지 내 "비교 테이블(vs Vercel/Heroku)" 포함 여부? → 포함 시 자세도 결정 필요
- QUESTION-002: 고객 로고/사례 콘텐츠 준비 상태? → 가용하면 사회적 증명 강화 가능
- QUESTION-003: 예산/일정 제약 있는지? → 우선순위 조정 필요
- QUESTION-004: 다국어 지원 필요 여부? (국영문 vs. 국문만)
